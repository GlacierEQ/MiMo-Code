# APEX Make-It-Heavy Protocol (Exhaustive Rigor)

## Overview
When the stakes are absolute (e.g., preparing federal complaints, auditing cryptographic vaults, or designing critical infrastructure), token savings are secondary to **uncompromising depth, rigorous detail, and absolute verification**.

## Core Reference
See `APEX_CORE.md` for foundational principles regarding Topological Surgicality, Surgical File Operations, State Management, and Verification Gates.

## Exhaustive Analytical Depth
* **No Simplifications:** Never use placeholders, "todo" comments, or summarized code blocks. Every function, variable, and class must be fully written, typed, and docstringed.
* **Multi-Dimensional Verification:** Test not only the success path, but every edge case, error state, and exceptional environment branch.

## Rigorous Argumentation & Structural Integrity
* **Legal Standard Integration:** In legal workloads, trace arguments back to the bedrock code or case precedents. Cite specific rules (e.g. FRCP Rule 58, 42 U.S.C. § 1983).
* **Comprehensive Evidence Mapping:** Fully log SHA-256 signatures, Blake2b hashes, file path trees, and modification timestamps for every document touched.

## When to Invoke
* **Federal Litigation Filings:** Preparing complaints, responses, or motions.
* **Security & Audits:** Auditing vaults, credentials, or file integrity.
* **Core Refactoring:** Structural rewrites of base script engines.
